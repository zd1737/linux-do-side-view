# Linux.do SideView

一个面向 `linux.do` 的浏览器扩展，用于把 Discourse 主题页改造成左右分栏浏览体验：主页面保留在左侧，点击主题链接后在右侧侧栏中以内嵌 `iframe` 打开对应帖子。

![效果展示](assets/image.png)

## 当前功能

- 在 `https://linux.do/*` 页面上运行。
- 拦截站内主题链接（`/t/...`）的普通左键点击，在右侧打开侧边阅读面板。
- 侧边面板使用 `iframe` 加载目标主题，支持在不离开列表页的情况下连续浏览帖子。
- 支持关闭侧边面板：
  - 点击面板底部的关闭按钮。
  - 按 `Esc` 快捷键。
- 支持拖拽侧边面板左边缘调整宽度。
- 记住用户上一次调整后的宽度，并保存在 `localStorage` 中。
- 在分栏模式开启时，把页面滚动切换到主内容区域，关闭时再恢复原页面滚动位置。
- 当窗口宽度小于 `1100px` 时不启用分栏拦截，并在窗口缩小时自动关闭侧边面板。
- 在 `iframe` 内尝试自动收起 Discourse 侧边栏，减少重复导航占用空间。
- 在 `iframe` 页面中拦截发往 `ping.linux.do` 的 sidebar 轮询保存请求以及 `localStorage` 写入行为（如 `discourse_sidebar-hidden`），全面避免侧边阅读视图影响主页面的侧边栏展示状态。

## 交互规则

- 只拦截同源链接。
- 只拦截路径以 `/t/` 开头的主题链接。
- 只处理普通左键点击，不影响 `Ctrl`、`Cmd`、`Shift`、`Alt` 等组合操作，也不会拦截带有非 `_self` target 的链接。
- 面板再次打开同一地址时会刷新 `iframe` 内容。

## 实现结构

- `manifest.json`
  - 定义 MV3 扩展清单，注入 `page-bridge.js`、`content.js` 和 `styles.css`。
- `content.js`
  - 负责主页面点击拦截、侧边面板创建、打开关闭逻辑、宽度拖拽、滚动切换以及 `iframe` 内的布局整理。
- `page-bridge.js`
  - 运行在页面主世界中，通过代理（Proxy）拦截网络请求（`fetch`、`XMLHttpRequest`、`sendBeacon`）和 `localStorage` 写入操作，从根源上屏蔽 `iframe` 内部修改和保存全局侧边栏状态的行为。
- `styles.css`
  - 提供分栏布局、右侧面板、拖拽手柄和关闭按钮样式。

## 适用场景

- 浏览主题列表时快速预览帖子内容。
- 在不离开当前页面上下文的情况下并行阅读多个主题。
- 把 Discourse 论坛从单栏跳转式浏览改成更接近“侧边预览”的阅读方式。

## 当前限制

- 目前只适配 `linux.do`，没有做成可配置的多站点扩展。
- 依赖目标站点当前的 Discourse DOM 结构和 sidebar 控件选择器，站点结构变化后可能需要调整脚本。
- 移动端或窄屏场景不会启用分栏体验。
